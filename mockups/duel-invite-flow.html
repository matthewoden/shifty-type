What changes under the hood

  • The opener plays before anyone joins. Today applyMove() rejects every move while the match waits for a second player. We start the match in the opener's turn with the second seat empty (tracked by a small awaitingOpponent flag), so creating and opening are one motion.
  • Join no longer forces the first move. When the friend joins an already-opened match, they step into their turn to answer your word — not an empty opening turn.
  • A real share button. navigator.share() (with a navigator.clipboard + code fallback) — the same machinery already used for the install link and seat link, pointed at /m/7XQ2.
  • A public preview endpoint. The landing screen needs the inviter's name and opening word without a token, so a small unauthenticated GET /api/match/:code/preview returns just those non-secret fields.
  • The tutorial remembers the invite. "Try the tutorial" stashes the pending code; on completion the exit routes back to the invite landing instead of Home.
  • The nudge copy gets honest. The "sat down — the chain starts with you" push contradicts the real turn order; this flow fixes it: the opener is told their friend arrived, and turn nudges follow real play.
  • Copy scrub. No player-facing "grip"; no emoji anywhere (Phosphor icons only, matching the shipped app).
